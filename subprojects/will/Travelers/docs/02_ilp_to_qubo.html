Step 1. The Integer Linear Program (ILP)
The optimization problem we want to solve, written as a binary program with linear objective and constraints.

f Objective Function

$$\max \sum_{m=1}^{M} \sum_{i=1}^{N} c_{i,m} \cdot x_{i,m}$$

Each $x_{i,m} \in \{0,1\}$ indicates whether coverage $i$ is included in package $m$. The coefficient $c_{i,m}$ is the contribution margin (profit) from that assignment.

$$c_{i,m} = \text{price}_i \times \text{margin\_pct}_i \times \underbrace{(1 - \delta_m)}_{\text{revenue factor}} \times \underbrace{(\text{take\_rate}_i \times \alpha_{i,m} \times (1 + \beta\,\delta_m))}_{\text{stimulated take rate}}$$

C Constraints (5 types)

#TypeFormulaCount/pkg
1 Mandatory families $\sum_{i \in F_k} x_{i,m} = 1$ 2
2 Optional families $\sum_{i \in F_k} x_{i,m} \le 1$ 7
3 Capacity $\sum_{i} x_{i,m} \le 7$ 1
4 Incompatibility $x_{i,m} + x_{j,m} \le 1$ 3
5 Dependency $x_{j,m} \le x_{i,m}$ 6

Step 2. Move Constraints into the Objective
For this challenge, we convert the constrained ILP into an unconstrained penalty objective. Other quantum formulations can preserve feasibility more directly, but penalty-QUBO is the bridge used here.

P The Penalty Method

$$\min_{x} \; \underbrace{-\sum c_{i,m}\, x_{i,m}}_{\text{negated objective (blue)}} + \underbrace{\lambda \sum_k P_k(x)}_{\text{penalty terms (red)}}$$

We flip to minimization (negate the objective) and add $\lambda \cdot P_k(x)$ for each constraint. Each $P_k(x) \ge 0$ is zero when the constraint is satisfied and positive when violated. The penalty weight $\lambda$ must be large enough that violating any constraint is never profitable.

This is one standard QUBO construction. It is powerful because it creates a single quadratic objective, but it is not the only way to represent constrained optimization on quantum hardware.

Objective (linear) terms    Penalty (quadratic) cross-terms    Slack variables

B Binary Algebra Shortcut

The reason this works so cleanly is that every decision variable is binary. For any binary bit,

$$x_i \in \{0,1\} \quad \Longrightarrow \quad x_i^2 = x_i$$

So when we expand a squared constraint like $(x_1 + x_2 - 1)^2$, the highest-order terms are only pairwise products such as $x_1 x_2$. That is exactly the quadratic structure required by a QUBO.

S Slack Variables for Inequalities

Inequalities like $\sum x_i \le K$ cannot be squared directly. We introduce slack variables $s \in \{0,1\}^{\lceil\log_2 K\rceil}$ that absorb the gap:

$$\sum_i x_i \le K \;\;\longrightarrow\;\; \sum_i x_i + \underbrace{s_0 + 2s_1 + \cdots}_{\text{slack (binary encoding)}} = K$$

The slack binary-encodes the integer gap $K - \sum x_i$, converting the inequality to an equality that can then be squared. Cost: $\lceil\log_2 K\rceil$ extra binary variables per inequality.

Mini Example: $x_4 + x_5 \le 1$

Rewrite it as $x_4 + x_5 + s = 1$ with one slack bit $s$.
If $(x_4, x_5) = (0,0)$, choose $s=1$ so the equality becomes $0 + 0 + 1 = 1$.
If $(x_4, x_5) = (1,0)$ or $(0,1)$, choose $s=0$.
If $(x_4, x_5) = (1,1)$, no binary $s$ can fix $1 + 1 + s = 1$, so the squared penalty becomes positive.

Key takeaway: Every constraint becomes a quadratic penalty. The cross-terms ($x_i x_j$) are the ZZ interactions that create entanglement in the quantum circuit. The linear terms shift the diagonal of the Q matrix. These are all the ingredients of a QUBO.

E The QUBO Energy

$$E(x) = \sum_i Q_{ii}\,x_i + \sum_{i < j} Q_{ij}\,x_i x_j$$

The minimum-energy bitstring is the optimal solution. Diagonal entries weight individual variable selections; off-diagonal entries encode pairwise interactions from the penalty terms.

A Anatomy of Q Entries

Diagonal entry $Q_{ii}$ (e.g., $Q_{11}$ for auto_liability_basic)
$Q_{11} = \underbrace{-c_{1,m}}_{\text{objective}} + \underbrace{\lambda(-1)}_{\text{mandatory penalty linear}}$

The objective wants to include high-$c$ coverages (negative = attractive).
The penalty's linear part also shifts the diagonal.
Off-diagonal entry $Q_{ij}$ (e.g., $Q_{12}$ for auto_liab_basic vs enhanced)
$Q_{12} = \underbrace{2\lambda}_{\text{mandatory family penalty}}$

Both are in the auto_base family. The $2\lambda$ penalty makes selecting both extremely costly — enforcing "pick exactly one."
Off-diagonal $Q_{45}$ (collision vs comprehensive)
$Q_{45} = \underbrace{2\lambda}_{\text{optional family penalty}}$

Both in auto_physical. Penalty enforces "at most one."
Slack column entry $Q_{4,s_1}$ (collision vs slack)
$Q_{4,s_1} = 2\lambda$

Slack variable $s_1$ absorbs the gap when neither collision nor comprehensive is selected.
Step 4. Block-Diagonal Structure
The full 200-variable QUBO decomposes into 10 independent 20-variable sub-problems -- one per package. No cross-package interactions exist.

B Full QUBO: 200 x 200 Variables

$N = 20$ coverages × $M = 10$ packages = 200 binary variables.
Click "Decompose" to see the block structure emerge.

I Why Block-Diagonal?

$$Q = \begin{pmatrix} Q^{(1)} & 0 & \cdots & 0 \\ 0 & Q^{(2)} & \cdots & 0 \\ \vdots & & \ddots & \\ 0 & 0 & \cdots & Q^{(M)} \end{pmatrix}$$

Every constraint involves only variables from a single package:

  • Mandatory/optional families: $\sum_{i \in F_k} x_{i,\textbf{m}}$ — only package $m$
  • Capacity: $\sum_i x_{i,\textbf{m}} \le 7$ — only package $m$
  • Incompatibility: $x_{i,\textbf{m}} + x_{j,\textbf{m}} \le 1$ — only package $m$
  • Dependency: $x_{j,\textbf{m}} \le x_{i,\textbf{m}}$ — only package $m$

There are no constraints linking $x_{i,m}$ to $x_{j,m'}$ for $m \ne m'$.

Practical impact for quantum algorithms:
Instead of one 200-variable logical problem register, the challenge splits into 10 independent 20-variable subproblems. That means circuit depth and gate count scale with $N$, not $N \times M$. Whether a full implementation fits on hardware still depends on slack bits, ancillas, compilation overhead, and the algorithm used.

F The Formula

$$\lambda \ge 2 \cdot \max_{i,m}(c_{i,m}) \cdot M$$

Treat this as a sufficient rule of thumb for this teaching example, not as a universal theorem for every QUBO. In practice, useful penalty scales are often instance-dependent.

This guarantees that violating any single constraint costs more than the maximum possible objective improvement from all variables combined.

For our mini-instance

$\max(c_{i,m})$ = 171.75 (auto_liability_basic)
$M$ = 1 (single package)
$\lambda \ge 2 \times 171.75 \times 1 =$ 343.50
Practical choice: $\lambda \approx 350$ to $500$

E Three Regimes

$\lambda$ too small — "Constraint Leakage"

Infeasible solutions (e.g., selecting both auto_liability_basic AND enhanced) have lower energy than feasible ones. The quantum algorithm finds constraint-violating "cheating" solutions that look optimal but are meaningless.

x = [1, 1, 1, 1, 1] ← "select everything" looks great but violates mandatory family!
$\lambda$ just right — "Clear Landscape"

All feasible solutions sit in a well-separated low-energy basin. The optimal feasible solution has the unique global minimum. Infeasible solutions are pushed to high energy. The quantum algorithm can distinguish good from great.

x = [1, 0, 1, 1, 0] ← basic + homeowners + collision = best feasible in this mini-instance
$\lambda$ too large — "Flat Landscape"

Penalty terms dominate, so all feasible solutions have nearly the same energy (the objective differences are tiny compared to $\lambda$). The quantum algorithm cannot tell the optimal solution apart from mediocre feasible ones.

E(optimal) = -529.91 + 0 ≈ E(any feasible) ← all look the same
Practical tip: Start with $\lambda = 3 \cdot \max(c_{i,m})$ for a single-package instance. If the feasibility rate is low, increase $\lambda$. If many feasible solutions score similarly, decrease $\lambda$. Finding the sweet spot is itself an optimization problem, and the best value can shift from one instance family to another.
