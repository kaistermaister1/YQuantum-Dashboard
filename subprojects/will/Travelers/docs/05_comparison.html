Tale of Two Algorithms

Read This Comparison in Three Passes

Undergraduates usually get tripped up because quantum-computing comparisons mix three different questions:

  • Idea: how the algorithm searches for good solutions.
  • Per-circuit resources: qubits, depth, and two-qubit gates for one run.
  • Total workflow cost: optimizer iterations for QAOA versus post-selection losses for DQI.

Key Metrics

  • Approximation ratio: quantum best / classical optimum.
  • Feasibility rate: fraction of shots that decode to valid solutions.
  • Post-selection rate: fraction of DQI shots with syndrome 0.

Evidence Status

QAOA is older and better studied. DQI is newer and promising, but its performance depends on the problem encoding, decoder choice, and noise model.

QAOA
VS
DQI
"Tune the circuit to find the answer"
Iteratively adjust parameters to improve solution quality
QAOA treats the quantum circuit as a parameterized function and uses a classical optimizer to find the best angles. Think of it as gradient descent on a quantum energy landscape.
Philosophy
"Let interference find the answer"
Quantum interference naturally amplifies correct solutions
DQI encodes the problem constraints into the circuit structure itself. Destructive interference suppresses wrong answers and constructive interference amplifies correct ones -- no tuning needed.
Gradient descent on a quantum landscape
Classical optimization of quantum expectation values
Just like training a neural network, QAOA adjusts parameters to minimize a cost function. The "landscape" is the expectation value of the cost Hamiltonian as a function of the circuit angles.
Analogy
Error correction in reverse
Parity checks filter invalid solutions via syndrome measurement
In quantum error correction, you use parity checks and a decoder to diagnose errors. DQI borrows that mathematical structure: it rewrites the optimization problem as parity checks, then uses interference and decoding to enrich syndrome-0 outcomes.
Variational (optimizer in the loop)
Requires many circuit evaluations to converge
Each iteration: (1) prepare circuit with current angles, (2) measure many shots, (3) estimate cost, (4) update angles via classical optimizer. Repeat until convergence or budget exhausted.
Approach
One-shot (no optimizer needed)
Single circuit execution, then post-selection
DQI builds the problem into a fixed circuit, so there is no variational parameter search in the loop. That avoids QAOA-style optimizer landscapes, but the overall workflow still depends on the instance encoding, decoder, and post-selection rate.
Cost layer (ZZPhase) + Mixer (Rx)
Repeat p times: more layers = better approximation
Cost layer applies e^{-i*gamma*C} using ZZPhase gates (native on this hardware platform). Mixer layer applies e^{-i*beta*B} using Rx rotations. The parameter p controls circuit depth and solution quality.
Circuit
Dicke + Phase + Syndrome + Decoder
Challenge pipeline: prepare, encode, compute syndrome, decode, measure
At a high level, DQI uses Fourier/interference ideas to reduce optimization to decoding. In this hackathon implementation, the teaching pipeline is Dicke state + phase encoding + syndrome computation + an optional decoder stage + final interference/measurement.
2p tunable angles
(gamma_1, beta_1, ..., gamma_p, beta_p)
For p=1: 2 parameters. For p=3: 6 parameters. More parameters can improve results but make optimization harder. The "barren plateau" phenomenon means gradients can vanish exponentially with qubit count.
Parameters
Zero tunable parameters!
Circuit is fully determined by the problem structure
The main contrast with QAOA is that there is no variational angle search. The circuit is determined by the transformed instance, although practical performance still depends on decoder design, ancilla overhead, and noise.
This challenge: penalty terms in the cost Hamiltonian
Add lambda * (constraint violation)^2 to the objective
In this hackathon implementation, QAOA handles constraints with penalties. More broadly, the QAOA literature also includes constrained mixers and feasible-subspace variants, so penalties are a design choice here, not a universal rule.
Constraints
Parity check matrix (mod-2 encoding)
Constraints encoded in binary matrix B
Constraints are transformed into a GF(2) parity-check system. Syndrome-0 means the transformed parity checks are satisfied. For industrial ILPs, that translation may require helper bits and gadget clauses, so students should think of this as an encoding pipeline, not a simple mod-2 shortcut.
All measurements are candidates
Every shot returns a bitstring that can be evaluated
QAOA uses every measurement -- no shots are discarded. Each bitstring is a candidate solution that can be checked against constraints and evaluated for cost. Best-of-K sampling is common.
Output
Post-select: only syndrome=0 kept
Shots with wrong syndrome are discarded
Only measurements where the syndrome register reads all zeros are kept. This can discard a large fraction of shots, so DQI trades away sample efficiency in exchange for concentrating probability on transformed-instance feasibility.
Well-studied, tunable, uses all shots
Large body of literature, flexible depth parameter
QAOA has been studied extensively since 2014 (Farhi, Goldstone, Gutmann). Performance guarantees exist for certain problems. The depth parameter p allows trading circuit depth for solution quality.
Key Strength
No optimization loop, interference-powered
Avoids variational angle search
DQI's fixed-circuit design removes the classical optimizer from the loop. That is an attractive feature, but whether it wins in practice is still an empirical question that depends on structure, noise, and post-selection.
Optimizer can get stuck
Barren plateaus, many evaluations needed
For large n, the cost landscape can become exponentially flat (barren plateau), making it impossible to find good gradients. Each optimizer step requires many circuit shots, driving up total cost.
Key Weakness
Lossy mod-2, throws away shots
Post-selection success rate can be very low
A naive mod-2 conversion can lose information, which is why the industrial DQI pipeline uses helper bits and gadgets rather than simply taking every constraint modulo 2. Even with a careful encoding, low post-selection can still make the effective sample cost large.
What is settled, and what is still open? QAOA is the more mature variational baseline. DQI is a newer decoding-based approach with strong structure-aware motivation, but it should be presented to students as a promising method to benchmark, not as a universally superior replacement.

What Judges Want to See

● Strong Submission

  • ✓ Ran both algorithms at 3+ problem sizes
  • ✓ Validated against classical baseline (PuLP)
  • ✓ Scaling plots with error bars from multiple seeds
  • ✓ Explained WHY algorithms differ at each scale
  • ✓ Exploited block-diagonal structure (M sub-problems)
  • ✓ Reported circuit resource counts (qubits, gates, depth)
  • ✓ Clean, reproducible code with clear documentation

● Common Mistakes

  • ✗ Only ran one algorithm (no comparison!)
  • ✗ No classical baseline -- cannot compute approx. ratio
  • ✗ Only tested at one problem size -- no scaling insight
  • ✗ Reported numbers without explaining what they mean
  • ✗ Ignored block-diagonal structure (wasted qubits)
  • ✗ No error bars or statistical analysis
