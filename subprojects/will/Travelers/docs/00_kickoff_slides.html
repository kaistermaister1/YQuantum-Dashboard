Two Algorithms. One Quantum Computer.
One Real Insurance Problem.

Which quantum approach scales better on P&C insurance product bundling?

QAOA

The variational approach: a classical optimizer tunes quantum circuit parameters iteratively. Well-studied but faces barriers at low depth.

DQI

The interference approach: quantum mechanics + coding theory find solutions in one shot. No optimization loop — just physics.

04 / CONSTRAINTS

Five Constraint Types Shape the Landscape

Mandatory Families

Σ x = 1: Exactly one from auto_base and property_base per package. Choosing basic vs enhanced changes which optional coverages are available.

Optional Families

Σ x ≤ 1: At most one from each of the 7 optional families.

Capacity: K = 7

No more than 7 coverages per package. Forces the optimizer to pick the most profitable subset.

Incompatible Pairs

3 pairs that can't coexist: gap ↔ floater, ALE ↔ earthquake, excess ↔ flood.

Dependencies

6 rules: flood/earthquake/ALE require homeowners; umbrella/excess require enhanced auto liability; floater requires condo.

The Interaction

Mandatory choice gates optional access. Pick renters instead of homeowners? No flood, earthquake, or ALE allowed. This is the complexity.

05 / THE BRIDGE TO QUANTUM

ILP → QUBO → Quantum Circuit

Constraints move into the objective as penalty terms

1. The ILP

maximize Σ c · x
subject to Ax ≤ b

→

2. Add Penalties

minimize −Σc·x
+ λΣ(Ax − b)²

→

3. QUBO Matrix Q

minimize xTQx
Q encodes objective + penalties

→

4. Quantum!

QAOA: cost Hamiltonian
DQI: max-XORSAT

Key Insight:

The QUBO is block-diagonal across packages. Each package is an independent sub-problem. You never need 200 qubits — you need 20.

06 / QAOA

QAOA: The Variational Approach

Classical optimizer + quantum circuit in a feedback loop

The Loop

  • Prepare all qubits in superposition (H gates)
  • Cost layer: Rz for objective + ZZPhase for penalties
  • Mixer layer: Rx rotations explore new solutions
  • Measure → compute energy → optimizer adjusts angles
  • Repeat 100–1000 times until convergence

Why This Hardware Fits QAOA

  • ZZPhase is native: each penalty term = 1 gate
  • All-to-all connectivity: no SWAP overhead
  • Rz is free: virtual gate, zero physical cost
  • Start with p = 1, try p = 2 if time permits

07 / DQI

DQI: The Interference Approach

Error correction in reverse — one shot, no optimization loop

The Pipeline

  • Dicke state: equal superposition of k-hot strings
  • Phase encoding: Rz(ci) on each variable qubit
  • Syndrome encoding: CNOT network from parity matrix B
  • Decoder stage: optional BP1-style decoding for feasibility steering
  • Hadamard + Measure: interfere and read out
  • Post-select: keep only syndrome = 0 results

Key Tradeoffs

  • No optimizer loop: run once, post-select, done
  • More qubits needed: n variables + m syndrome qubits
  • Lossy encoding: mod-2 reduces constraints
  • Post-selection cost: ~10–30% of shots kept
  • Based on LDPC coding theory (same math as 5G!)

08 / HEAD TO HEAD

QAOA vs DQI

Aspect QAOA DQI
Philosophy Tune the circuit VS Let interference work
Parameters 2p angles to optimize VS Zero tunable parameters
Circuit evaluations 100–1000 iterations VS Single run
Constraints Penalty terms (λ) VS Parity check matrix (mod-2)
Measurement usage Uses all shots VS Discards ~70–90% (post-select)
Qubits (per sub-problem) N = 5–20 VS N + m ≈ 2N
Note:

Read this table as a comparison of workflows, not a proof that one algorithm is universally better. QAOA is the more mature variational method; DQI is a newer decoder-based approach that can behave very differently depending on the encoding and noise model.

13 / TIPS FOR SUCCESS

Tips for Success

  • Start small: Get both algorithms working on n=10 before scaling up
  • Classical baselines first: Solve with PuLP so you know the optimum
  • Exploit the block structure: Decompose into M independent sub-circuits
  • ZZPhase is your friend: Each penalty maps naturally to a native ZZ gate
  • Tune λ carefully: Start with λ = 3 × max(c) and adjust
  • Mandatory choice matters: Which coverage from auto_base/property_base you pick gates the optional coverages via dependencies
  • Read the starter code: src/insurance_model.py, the learning guides, and the comparison notebook
